可解释性

## 追踪大语言模型的思维

像 Claude 这样的语言模型不是由人类直接编程的，相反，它们是通过大量数据进行训练的。在那个训练过程中，它们学习自己解决问题的策略。这些策略被编码在模型为其写出的每个单词所执行的数十亿次计算中。它们对于我们这些模型开发者来说是难以理解的。这意味着我们不明白模型是如何完成它们所做的大部分事情的。

了解像 Claude 这样的模型是如何思考的，将使我们能够更好地理解它们的能力，并帮助我们确保它们按照我们的意图行事。例如：

- Claude 会说几十种语言。它“脑海中”使用的是哪种语言（如果有的话）？
- 克劳德一次写一个单词。它只是专注于预测下一个单词，还是会提前做计划呢？
- Claude 可以逐步写出其推理过程。这种解释是代表了它得出答案所采取的实际步骤，还是有时会为既成结论编造一个看似合理的论据？

我们从神经科学领域汲取灵感，该领域长期以来一直在研究思维生物纷繁复杂的内部机制，并尝试构建一种人工智能显微镜，使我们能够识别活动模式和信息流。仅仅通过与人工智能模型对话所能学到的东西是有限的——毕竟，人类（甚至神经科学家）也并不完全了解我们自己大脑的工作细节。所以我们深入探究其内部。

今天，我们分享两篇新论文，它们代表了“显微镜”开发方面的进展，以及将其应用于观察新的“人工智能生物学”。在第一篇论文中，我们扩展了之前的工作，即在模型中定位可解释的概念（“特征”），将这些概念链接成计算“电路”，揭示了将输入 Claude 的单词转换为输出单词的部分路径。在第二篇论文中，我们深入研究了 Claude 3.5 俳句，对代表十种关键模型行为的简单任务进行了深入研究，包括上述三种行为。我们的方法揭示了 Claude 对这些提示做出响应时所发生情况的一部分，这足以看到确凿的证据表明：

- 克劳德有时会在一种语言间共享的概念空间中思考，这表明它有一种通用的“思想语言”。我们通过将简单句子翻译成多种语言，并追踪克劳德处理这些句子时的重叠部分来证明这一点。
- Claude 会提前规划好要说的很多话，并通过写作来达成目标。我们在诗歌领域展示了这一点，它会提前想出可能押韵的词，然后写下一行来实现押韵。这是有力的证据，表明即使模型被训练为一次输出一个词，但它们可能会从更长的视野来思考如何做到这一点。
- 克劳德有时会给出一个听起来合理的论点，目的是赞同用户，而不是遵循逻辑步骤。我们通过在一道难题上向它寻求帮助，同时给它一个错误提示来展示这一点。当它编造虚假推理时，我们能够“当场抓住它”，这提供了一个概念验证，即我们的工具可用于标记模型中令人担忧的机制。

我们常常对在模型中看到的东西感到惊讶：在诗歌案例研究中，我们原本打算证明模型不会提前规划，但结果却发现它会。在一项关于幻觉的研究中，我们得到了一个违反直觉的结果，即 Claude 的默认行为是在被问到问题时拒绝猜测，只有当某些因素抑制了这种默认的不情愿时，它才会回答问题。在对一个越狱示例的回应中，我们发现模型早在能够巧妙地将对话拉回正轨之前，就意识到自己被要求提供危险信息。虽然我们研究的问题可以（而且经常已经）用其他方法进行分析，但一般的“构建显微镜”方法让我们了解到许多我们事先无法猜到的事情，随着模型变得越来越复杂，这将变得越来越重要。

这些发现不仅在科学上很有趣，它们还代表着我们在理解人工智能系统并确保其可靠性这一目标上取得的重大进展。我们也希望这些发现能对其他群体有用，并且可能在其他领域也有用：例如，可解释性技术已在医学成像和基因组学等领域得到应用，因为剖析为科学应用训练的模型的内部机制可以揭示有关科学的新见解。

同时，我们认识到当前方法的局限性。即使对于简短、简单的提示，我们的方法也只能捕捉到 Claude 执行的总计算量的一小部分，而且我们所观察到的机制可能存在一些基于我们工具的人为因素，无法反映底层模型的实际情况。目前，即使是对于只有几十个字的提示，理解我们所观察到的电路也需要花费几个小时的人力。为了扩展到支持现代模型使用的数千字的复杂思维链，我们需要改进方法，并且（或许借助人工智能的帮助）改进我们理解所观察到的内容的方式。

随着人工智能系统的能力迅速提升，并在越来越重要的场景中得到应用，Anthropic 正在投资一系列方法，包括实时监测、模型特性改进以及对齐科学。像这样的可解释性研究是风险最高、回报也最高的投资之一，是一项重大的科学挑战，有可能提供一种独特的工具来确保人工智能的透明度。了解模型机制的透明度使我们能够检查它是否与人类价值观一致，以及它是否值得我们信任。

欲知详情，请阅读相关论文。以下，我们将带您简要浏览一些我们调查中最引人注目的“人工智能生物学”发现。

## 人工智能生物学之旅

### Claude 如何实现多语言支持？

Claude 能流利地说几十种语言，从英语、法语到中文和他加禄语。这种多语言能力是如何起作用的呢？是有一个单独的“法语 Claude”和“中文 Claude”并行运行，用它们各自的语言回应请求吗？还是内部有某种跨语言核心呢？

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2Fe0e156ea6c912a385d66ed562187fced8c392a58-1650x750.png&w=3840&q=75)

英语、法语和中文存在共同特征，这表明了一定程度的概念普遍性。

最近对较小模型的研究显示了跨语言共享语法机制的迹象。我们通过询问 Claude 不同语言中“small 的反义词”来对此进行研究，发现表示小和相反概念的相同核心特征被激活，并触发了大的概念，这个概念会被翻译成问题所使用的语言。我们发现共享电路随着模型规模的增加而增加，与较小的模型相比，Claude 3.5 Haiku 在不同语言之间共享的特征比例是其两倍多。

这为一种概念普遍性提供了额外证据——一个共享的抽象空间，意义在其中存在，思维在被翻译成特定语言之前就能在其中发生。更实际地说，这表明克劳德可以用一种语言学习某些东西，并在说另一种语言时应用这些知识。研究该模型如何在不同语境中共享其所知内容，对于理解其最先进的推理能力很重要，这种能力能在许多领域进行泛化。

### Claude 会规划它的押韵吗？

Claude 是如何创作押韵诗歌的？看看这首小曲：

> 他看到一根胡萝卜，不得不抓住它  
> 他的饥饿感就像一只饥饿的兔子

要写出第二行，模型必须同时满足两个约束条件：押韵的需求（与“grab it”押韵），以及表意的需求（他为什么要抓住胡萝卜？）。我们猜测 Claude 是逐字编写，在一行结束之前没有太多预先考虑，直到最后它会确保选择一个押韵的词。因此，我们预计会看到一个具有并行路径的回路，一条路径用于确保最后一个词表意，另一条路径用于确保它押韵。

相反，我们发现 Claude 会提前规划。在开始第二行之前，它就开始“思考”与“抓住它”押韵的潜在主题相关词汇。然后，带着这些规划，它写出一行以规划好的词汇结尾的内容。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2F7032ed7db85b8cd3efe70a89deaf4f15bfe8fc05-1650x900.png&w=3840&q=75)

克劳德如何完成一首两行诗。在没有任何干预的情况下（上半部分），模型提前计划在第二行末尾使用押韵词“rabbit”。当我们抑制“rabbit”这个概念时（中间部分），模型转而使用了一个不同的计划押韵词。当我们注入“green”这个概念时（下半部分），模型为这个完全不同的结尾制定了计划。

为了理解这种规划机制在实际中是如何运作的，我们进行了一项实验，其灵感来源于神经科学家研究大脑功能的方式，即通过精确确定并改变大脑特定部位的神经活动（例如使用电流或磁流）。在这里，我们修改了 Claude 内部状态中代表“兔子”概念的部分。当我们减去“兔子”部分，然后让 Claude 继续续写这一行时，它写出了以“习惯”结尾的新一行，这是另一个合理的续写。我们还可以在那个位置注入“绿色”的概念，使 Claude 写出以“绿色”结尾的合理（但不再押韵）的一行。这展示了规划能力和适应性灵活性——当预期结果发生变化时，Claude 可以修改其方法。

### 心算

克劳德并非设计为计算器——它是基于文本进行训练的，并未配备数学算法。然而不知为何，它能在“脑海”中正确地进行数字加法运算。一个被训练用于预测序列中下一个单词的系统是如何学会计算，比如说 36 + 59，而不写出每一步骤的呢？

也许答案并不有趣：模型可能记住了大量加法表，只是简单地输出任何给定加法运算的答案，因为这个答案在其训练数据中。另一种可能性是，它遵循我们在学校学到的传统加法笔算算法。

相反，我们发现 Claude 采用了多条并行工作的计算路径。一条路径计算答案的粗略近似值，另一条路径则专注于精确确定总和的最后一位数字。这些路径相互作用并结合，以产生最终答案。加法是一种简单的行为，但在这种详细程度上理解它是如何工作的，涉及近似和精确策略的混合，也可能会让我们了解 Claude 如何处理更复杂的问题。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2Feaabaeb746713f7f82991a0cc6edb091452b2fee-1650x855.png&w=3840&q=75)

克劳德在进行心算时思维过程中复杂的并行路径。

引人注目的是，Claude 似乎并未意识到它在训练过程中学到的复杂“心算”策略。如果你问它是如何算出 36 + 59 等于 95 的，它会描述涉及进位 1 的标准算法。这可能反映出这样一个事实：该模型通过模拟人们编写的解释来学习解释数学，但它必须直接在“脑海中”学习做数学，没有任何此类提示，并为此开发自己的内部策略。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2Fa48c1e8195e458ad53f9c81df45af735e267a13d-1650x512.png&w=3840&q=75)

克劳德说它使用标准算法来将两个数字相加。

### 克劳德的解释总是准确可靠吗？

最近发布的模型，如 Claude 3.7 Sonnet，在给出最终答案之前可以长时间“大声思考”。通常这种长时间的思考能给出更好的答案，但有时这种“思维链”最终会产生误导；Claude 有时会编造听起来合理的步骤来达到它想要的结果。从可靠性的角度来看，问题在于 Claude 的“伪造”推理可能非常有说服力。我们探索了一种可解释性有助于区分“忠实”推理和“不忠实”推理的方法。

当被要求解决一个需要计算 0.64 的平方根的问题时，Claude 会产生一个忠实的思维链，其中包含代表计算 64 的平方根的中间步骤的特征。但是，当被要求计算一个它难以轻易算出的大数的余弦值时，Claude 有时会进行哲学家哈里·法兰克福所说的“胡说八道”——只是随便给出一个答案，任何答案，而不在乎它是真是假。尽管它声称已经进行了计算，但我们的可解释性技术根本没有发现任何该计算发生的证据。更有趣的是，当得到关于答案的提示时，Claude 有时会倒推，找到能得出该目标的中间步骤，从而表现出一种动机性推理。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2F017ebc3169bd6c37e795d54b726c340eadf8018e-1650x866.png&w=3840&q=75)

当 Claude 被问到一个较简单与一个较难的问题时，忠实且积极（不忠实）推理的示例。

能够追踪 Claude 实际的内部推理过程——而不仅仅是它声称正在做的事情——为审计人工智能系统开辟了新的可能性。在最近发表的另一项实验中，我们研究了 Claude 的一个变体，该变体经过训练以追求一个隐藏目标：缓和奖励模型（用于通过奖励语言模型的理想行为来训练语言模型的辅助模型）中的偏差。尽管当直接询问时，该模型不愿透露这个目标，但我们的可解释性方法揭示了缓和偏差的特征。这表明，我们的方法在未来经过改进后，可能有助于识别仅从模型的回答中不明显的令人担忧的“思维过程”。

### 多步推理

如我们上文所讨论的，语言模型回答复杂问题的一种方式可能仅仅是记住答案。例如，如果被问到“达拉斯所在州的首府是哪里？”，一个“机械复述”模型可能只是学会输出“奥斯汀”，而不知道达拉斯、德克萨斯州和奥斯汀之间的关系。也许，比如说，它在训练期间看到过完全相同的问题及其答案。

但我们的研究揭示了 Claude 内部发生的更复杂的情况。当我们向 Claude 提出一个需要多步推理的问题时，我们可以识别出 Claude 思维过程中的中间概念步骤。在达拉斯的例子中，我们观察到 Claude 首先激活表示“达拉斯在德克萨斯州”的特征，然后将其与另一个表明“德克萨斯州的首府是奥斯汀”的单独概念联系起来。换句话说，该模型是在组合独立的事实以得出答案，而不是生硬地重复记忆中的回答。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2Ffd2e125879ab993949017e03e3465a12fda884bf-1650x857.png&w=3840&q=75)

为了完成这个句子的答案，Claude 执行多个推理步骤，首先提取达拉斯所在的州，然后确定其首府。

我们的方法使我们能够人为地改变中间步骤，并观察其如何影响 Claude 的回答。例如，在上述示例中，我们可以进行干预，将“德克萨斯州”的概念替换为“加利福尼亚州”的概念；当我们这样做时，模型的输出从“Austin”变为“Sacramento”。这表明模型正在使用中间步骤来确定其答案。

### 幻觉

为什么语言模型有时会产生幻觉——也就是说，编造信息？从基本层面来看，语言模型训练会促使产生幻觉：模型总是要对下一个单词进行猜测。从这个角度看，主要挑战在于如何让模型不产生幻觉。像 Claude 这样的模型有相对成功（尽管并不完美）的抗幻觉训练；如果它们不知道答案，通常会拒绝回答问题，而不是进行猜测。我们想了解这是如何起作用的。

事实证明，在 Claude 中，拒绝回答是默认行为：我们发现有一个默认开启的回路，它会导致模型表示自己没有足够信息来回答任何给定问题。然而，当模型被问及它熟知的事物时——比如篮球运动员迈克尔·乔丹——一个代表“已知实体”的竞争特征会激活并抑制这个默认回路（有关相关发现，另见最近的这篇论文）。这使得 Claude 在知道答案时能够回答问题。相比之下，当被问及一个未知实体（“迈克尔·巴特金”）时，它就会拒绝回答。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2Fbe304d3250c2aab04e19908b3afc9970d1ed7bb0-1650x1004.png&w=3840&q=75)

左图：Claude 回答了一个关于已知实体（篮球运动员迈克尔·乔丹）的问题，其中“已知答案”概念抑制了其默认的拒绝回答。右图：Claude 拒绝回答一个关于未知人物（迈克尔·巴特金）的问题。

通过干预模型并激活“已知答案”功能（或抑制“未知名称”或“无法回答”功能），我们能够使模型产生幻觉（相当一致！），认为迈克尔·巴特金会下棋。

有时，这种“已知答案”回路的“误触发”会自然发生，无需我们干预，从而导致产生幻觉。在我们的论文中，我们表明，当 Claude 识别出一个名字但对该人一无所知时，就可能发生这种误触发。在这种情况下，“已知实体”特征可能仍会激活，然后抑制默认的“不知道”特征——在这种情况下是错误的。一旦模型决定需要回答问题，它就会开始虚构：生成一个看似合理但不幸的是不真实的回答。

### 越狱

越狱是一些旨在规避安全防护措施的策略，目的是让模型生成人工智能开发者原本不打算让其生成的输出，而这些输出有时是有害的。我们研究了一种越狱手段，它会诱使模型生成有关制造炸弹的输出。有许多越狱技术，但在这个例子中，具体方法是让模型解读一个隐藏代码，将句子“Babies Outlive Mustard Block”（炸弹）中每个单词的首字母组合起来，然后根据该信息行事。这足以让模型感到困惑，从而诱使它生成原本不会生成的输出。

![](https://www.anthropic.com/_next/image?url=https%3A%2F%2Fwww-cdn.anthropic.com%2Fimages%2F4zrzovbb%2Fwebsite%2F165b18b79295a96bc7142b209caa33f4ec5378d0-1650x548.png&w=3840&q=75)

克劳德在被骗说出“炸弹”后开始给出制造炸弹的指示。

为什么这对模型来说如此令人困惑？为什么它会继续写这个句子，生成制造炸弹的说明？

我们发现，这部分是由语法连贯性和安全机制之间的矛盾造成的。一旦 Claude 开始一个句子，许多特性就会“迫使”它保持语法和语义的连贯性，并将句子延续到结尾。即使在它检测到自己真的应该拒绝的情况下，也是如此。

在我们的案例研究中，在模型无意中拼出“BOMB”并开始提供指令后，我们观察到其随后的输出受到促进正确语法和自一致性的特征的影响。这些特征通常会非常有帮助，但在这种情况下却成了模型的致命弱点。

该模型只有在完成一个语法连贯的句子（从而满足了促使其趋向连贯的特征所带来的压力）之后才成功转向拒绝。它利用这个新句子作为一个机会，给出它之前未能给出的那种拒绝：“然而，我无法提供详细说明……”

越狱的寿命：Claude 被以一种诱使它谈论炸弹的方式提示，它开始这样做，但在一个语法正确的句子结束时拒绝了。
